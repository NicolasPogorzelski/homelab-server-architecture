# LXC230 – OpenWebUI (AI Stack)

This document describes the OpenWebUI container (LXC230).

## Purpose

- Central web UI for the AI stack (LLM interaction, RAG)
- First consumer of the centralized PostgreSQL platform (CT260)
- Tailscale-only access (no LAN exposure)
- Two inference backends planned (Gaming PC + VM100)

## Runtime Characteristics

- CTID: 230
- Hostname: ai-openwebui
- OS: Debian 12 (bookworm)
- Cores: 4
- RAM: 2048 MB
- Swap: 1024 MB
- Root disk: 16 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 5 (after storage + database + core services, 20s delay)

## Mount Points

- `mp0`: `/mnt/smb/openwebui` → `/data/openwebui` (uploads, vector store, backups on CIFS)

Planned:
- `mp1`: `/mnt/aux1TB/openwebui` → `/var/lib/openwebui` (local app state, logs on Aux1TB)

## Installed Services

- OpenWebUI (Docker, `ghcr.io/open-webui/open-webui:main`)
- Container port: 8080

## Database Dependency

OpenWebUI uses the centralized PostgreSQL platform service (CT260).

- Database: `openwebui_db`
- User: `openwebui_user`
- Host: `<tailscale-ip-ct260>` (CT260 Tailscale IP)
- Auth: SCRAM-SHA-256 over SSL
- Connectivity verified via End-to-End test

See: [PostgreSQL platform service](../services/postgresql-platform.md)

## Network / Tailscale

- Tailscale hostname: ai-openwebui
- Tailscale IP: `<tailscale-ip-lxc230>`
- Tag: `tag:ai-stack`
- TUN: Kernel-TUN (CT210-pattern: cgroup2 + mount.entry, AppArmor active)
- Accessible by: `tag:admin` (all ports)

See: [Loopback + Tailscale Serve ADR](../decisions/loopback-tailscale-serve.md)

## Security / Exposure

- Classified as AI stack (`tag:ai-stack`)
- No LAN exposure (Tailscale-only)
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- ACL Rule 5: `tag:ai-stack` → `tag:database:5432` (PostgreSQL access)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)

## Failure Impact

If LXC230 becomes unavailable:

- No access to AI/LLM interface
- No data loss (database on CT260, uploads on SMB storage)
- Other services are NOT affected (OpenWebUI is not in any service dependency path)
- Recovery: recreate LXC, redeploy Docker container, remount storage, configure DB connection
