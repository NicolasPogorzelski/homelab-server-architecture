## Purpose

- Central web UI for the AI stack (LLM interaction, RAG)
- First consumer of the centralized PostgreSQL platform (CT260)
- Tailscale-only access (no LAN exposure)
- Two inference backends planned (Gaming PC + VM100)

- Hostname: ai-openwebui

- Cores: 4
- RAM: 2048 MB
- Swap: 1024 MB
- Root disk: 16 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 6 (after database platform CT260, 20s delay)

## Security / Exposure

- Classified as AI stack (`tag:ai-stack`)
- No LAN exposure (Tailscale-only)
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- ACL Rule 5: `tag:ai-stack` â†’ `tag:database:5432` (PostgreSQL access)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)

## Failure Impact

If LXC230 becomes unavailable:

- No access to AI/LLM interface
- No data loss (database on CT260, uploads on SMB storage)
- Other services are NOT affected (OpenWebUI is not in any service dependency path)
- Recovery: recreate LXC, redeploy Docker container, remount storage, configure DB connection
