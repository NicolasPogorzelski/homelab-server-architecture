# LXC200 – Monitoring

This document describes the dedicated monitoring container (LXC200).

## Purpose

- Prometheus metrics collection
- Grafana dashboards and visualization
- Node Exporter system metrics
- Observability independent from application services

## Runtime Characteristics

- CTID: 200
- Hostname: monitoring
- OS: Debian 12 (bookworm)
- Cores: 2
- RAM: 1024 MB
- Swap: 512 MB
- Root disk: 8 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 2 (after storage, 20s delay)

## Mount Points

- `mp0`: `/mnt/aux1TB/monitoring` → `/data` (persistent monitoring data + Docker engine data on Aux1TB)

Subdirectory layout under `mp0`:

    /data/
    ├── prometheus/    → Prometheus TSDB
    ├── grafana/       → Grafana state
    ├── alertmanager/  → Alertmanager state
    ├── containerd/    → containerd image store + snapshots
    └── docker-data/   → Docker named volumes, network metadata

## LXC Configuration

Tailscale TUN configuration (CT210-pattern, see [DD#13](../decisions/design-decisions.md#13-standardized-tun-configuration-for-tailscale-capable-lxcs)):

    lxc.cgroup2.devices.allow: c 10:200 rwm
    lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file

Note: `lxc.apparmor.profile: unconfined` and empty `lxc.cap.drop` were removed
during the `chore/tailscale-tun-cleanup` — both were debugging leftovers
inconsistent with the least-privilege model.

## Installed Services

- Prometheus (`127.0.0.1:9090`)
- Grafana (`127.0.0.1:3000`)
- Node Exporter (`127.0.0.1:9100`)
- Alertmanager (`127.0.0.1:9093`)

All services bind to loopback only. Remote access via Tailscale.

## Network / Tailscale

- Tailscale hostname: monitoring
- Tag: `tag:monitoring`
- Accessible by: `tag:admin` (all ports)

### Tailscale Serve

| Serve Port (HTTPS) | Backend (HTTP) | Service |
|---|---|---|
| 443 | 127.0.0.1:3000 | Grafana |
| 9443 | 127.0.0.1:9090 | Prometheus |
| 9093 | 127.0.0.1:9093 | Alertmanager |
```bash
tailscale serve --bg --https=443 http://127.0.0.1:3000
tailscale serve --bg --https=9443 http://127.0.0.1:9090
tailscale serve --bg --https=9093 http://127.0.0.1:9093
```

URLs:
- `https://monitoring.<tailnet-id>.ts.net` (Grafana)
- `https://monitoring.<tailnet-id>.ts.net:9443` (Prometheus)
- `https://monitoring.<tailnet-id>.ts.net:9093` (Alertmanager)

See: [Loopback + Tailscale Serve ADR](../decisions/loopback-tailscale-serve.md)

## Security / Exposure

- All services bind to loopback only
- No LAN exposure
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)
- LXC TUN config follows CT210-pattern (DD#13); no AppArmor overrides

## Failure Impact

If LXC200 becomes unavailable:

- No metrics collection or alerting
- Running services are NOT affected (monitoring is not in the data path)
- Loss of observability until restored
- Recovery: recreate LXC, redeploy monitoring stack, restore Grafana dashboards
