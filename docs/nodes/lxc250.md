# LXC250 – DevOps Workstation

This document describes the central DevOps management node (LXC250).

All Git operations, code editing, and infrastructure-as-code work happen exclusively on this node.

## Purpose

- Central Git repository management (push/pull to GitHub)
- Code editing (Neovim, VS Code Remote SSH)
- Ansible control node (planned)
- IaC development and testing
- SSH jump point for administrative access to other nodes

## Runtime Characteristics

- Unprivileged Debian 12 LXC
- CTID: 250
- Hostname: `devops`
- Resources: 2 cores, 2048 MB RAM, 512 MB swap
- Disk: 8 GB on local-lvm
- Nesting enabled (required for Tailscale)
- Onboot: yes

## Installed Software

- Git (GitHub SSH authentication via ED25519 key)
- OpenSSH server (key-based auth from admin devices)
- Tailscale (tag:admin)
- sudo
- curl

Planned additions:
- Ansible
- tmux
- Neovim

## User Model

- Service user: `devops` (UID 1000, member of sudo group)
- Root access: via sudo
- SSH key authentication configured for admin devices

## Repository Layout

Git repositories are stored under:

- `/home/devops/git/homelab-server-architecture`

## Network / Tailscale

- Tailscale IP: assigned dynamically (check `tailscale status`)
- Tag: `tag:admin`
- ACL permissions: full access to tag:admin, tag:tier0, tag:tier1, tag:tier2, tag:storage

## Security / Exposure

- SSH binds to all interfaces (port 22)
- Authentication: SSH key only (ED25519) from admin-tagged devices
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)

## Access Model (Zero Trust)

- Administrative node with broad access to all tiers
- No service provider role (does not expose ports to other nodes)
- Outbound-only access pattern: SSH to other nodes, Git push/pull to GitHub
- Network segmentation enforced via Tailscale ACL

## Storage

- No external storage mounts
- All data is local to the container (Git repos, config files, SSH keys)
- No CIFS/SMB dependencies

## Failure Impact

If LXC250 becomes unavailable:

- No infrastructure changes can be pushed (Git, Ansible)
- Running services are NOT affected (devops is not in the data path)
- Recovery: recreate LXC, re-clone repos from GitHub, restore SSH keys

This node is operationally non-critical for service availability.
