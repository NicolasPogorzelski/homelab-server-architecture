# LXC211 – Paperless-ngx

This document describes the Paperless-ngx document management container (LXC211).

## Purpose

- Self-hosted document management system with OCR
- Automatic classification, tagging, and full-text search
- Security-critical service (handles personal documents)

## Runtime Characteristics

- CTID: 211
- Hostname: paperless
- OS: Debian 12 (bookworm)
- Cores: 2
- RAM: 2048 MB
- Swap: 512 MB
- Root disk: 10 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 3 (after storage + monitoring, 20s delay)

## Mount Points

- `mp0`: `/mnt/smb/paperless` → `/data/paperless` (document storage on MergerFS: originals, archive, thumbnails, consumption, export)
- `mp1`: `/mnt/aux1TB/paperless` → `/var/lib/paperless` (local runtime state: logs, app data)

Design rationale: Document files live on MergerFS (parity-protected, expandable).
Runtime state (logs, temporary data) uses local block storage for performance.
No database files on SMB (PostgreSQL runs in CT260).

## Installed Services

- Paperless-ngx (`127.0.0.1:8000`)
- Redis (sidecar container, task queue + caching)
- Gotenberg (document conversion)
- Apache Tika (content extraction)

All services run as Docker containers via Compose.

## LXC Configuration

Tailscale TUN configuration (CT210-pattern, see [DD#13](../decisions/design-decisions.md#13-standardized-tun-configuration-for-tailscale-capable-lxcs)):

    lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file

## Security / Exposure

- Classified as tier 1 (security-critical)
- Loopback-only binding (`127.0.0.1:8000`)
- No LAN exposure
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)
- LXC TUN config follows CT210-pattern (DD#13)

## Database Dependency

Paperless-ngx uses the centralized PostgreSQL platform service (CT260).

- Database: `paperless_db`
- User: `paperless_user`
- Connection: Tailscale IP of CT260, port 5432 (TLS required)
- pg_hba: `hostssl` entry scoped to CT211 Tailscale IP `/32`

See: [PostgreSQL Platform Service](../services/postgresql-platform.md)

## Failure Impact

If LXC211 becomes unavailable:

- No document ingestion or search
- Existing documents on SMB storage are not affected
- PostgreSQL data (metadata, tags, classifications) remains intact in CT260
- Recovery: recreate LXC, redeploy Docker stack, remount storage, verify DB connectivity

If PostgreSQL (CT260) becomes unavailable:

- Paperless cannot start or process documents
- Document files on SMB remain intact
- Recovery: restore CT260 first, then restart Paperless containers
