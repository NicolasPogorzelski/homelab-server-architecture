# LXC220 – Calibre-Web

This document describes the Calibre-Web container (LXC220).

## Purpose

- Web-based e-book library management
- Read-only consumer of the Books storage share

## Runtime Characteristics

- CTID: 220
- Hostname: calibreweb
- OS: Debian 12 (bookworm)
- Cores: 2
- RAM: 1024 MB
- Swap: 512 MB
- Root disk: 8 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 5 (after core services, 20s delay)

## Mount Points

- `mp0`: `/mnt/smb/books` → `/books` (read-only book library from SMB storage)

## Network / Tailscale

- Tailscale hostname: calibreweb
- Tag: `tag:tier2`
- Accessible by: `tag:admin` (all ports), `tag:tier0` (all ports), `tag:tier2` (all ports), `tag:client` (port 443), `tag:untrusted` (port 443)

## Security / Exposure

- Loopback-only binding (`127.0.0.1:8083`)
- No LAN exposure
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)

## Failure Impact

If LXC220 becomes unavailable:

- No access to the e-book library via web interface
- No data loss (library is read-only from SMB storage)
- Recovery: recreate LXC, redeploy Docker container, remount storage
