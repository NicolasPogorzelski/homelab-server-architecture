# LXC220 – Calibre-Web

This document describes the Calibre-Web container (LXC220).

## Purpose

- Web-based e-book library management
- Read-only consumer of the Books storage share

## Runtime Characteristics

- CTID: 220
- Hostname: calibreweb
- OS: Debian 12 (bookworm)
- Cores: 2
- RAM: 1024 MB
- Swap: 512 MB
- Root disk: 8 GB (local-lvm)
- Unprivileged: yes
- Nesting: enabled
- Onboot: yes
- Startup order: 5 (after core services, 20s delay)

## Mount Points

- `mp0`: `/mnt/smb/books` → `/books` (read-only book library from SMB storage; Calibre-Web web UI)
- `mp1`: `/mnt/aux1TB/calibreweb` → `/var/lib/calibreweb` (local config/state on the aux 1 TB disk)
- `mp2`: `/mnt/smb/books-rw` → `/books-rw` (read-write view of the same Books share; used **only** by the auto-import job — see [Configuration Management](#configuration-management))

## LXC Configuration

Tailscale TUN configuration (CT210-pattern, see [DD#13](../decisions/design-decisions.md#13-standardized-tun-configuration-for-tailscale-capable-lxcs)):

    lxc.cgroup2.devices.allow: c 10:200 rwm
    lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file

## Network / Tailscale

- Tailscale hostname: calibreweb
- Tag: `tag:tier2`
- Accessible by: `tag:admin` (all ports), `tag:tier0` (all ports), `tag:tier2` (all ports), `tag:client` (port 443), `tag:untrusted` (port 443)

tailscale serve --bg --https=443 http://127.0.0.1:8083

## Security / Exposure

- Loopback-only binding (`127.0.0.1:8083`)
- No LAN exposure
- No public exposure / no router port forwarding
- Remote access provided exclusively via Tailscale (identity-based overlay)
- Network policy enforced via Tailscale ACL (node tags + ACL JSON)
- See: [docs/platform/tailscale-acl.md](../platform/tailscale-acl.md)
- LXC TUN config follows CT210-pattern (DD#13)

## Failure Impact

If LXC220 becomes unavailable:

- No access to the e-book library via web interface
- No data loss (library is read-only from SMB storage via `mp0`)
- Auto-import stops; dropped files simply accumulate in `/books-rw/_import` until the node is back (the import job no-ops when the `mp2` rw mount is absent)
- Recovery: recreate LXC, redeploy Docker container, remount storage

## Configuration Management

Managed via Ansible from LXC250.

- **Stack path:** `/srv/calibreweb/` — active Compose stack; intentionally outside the repo clone
- **Playbook:** `ansible/playbooks/apt-upgrade.yml` — rolling apt upgrade (`serial: 1`)
- **Role:** `ansible/roles/node_exporter/` — deploys node_exporter binary + systemd unit
- **Role:** `ansible/roles/ssh-hardening/` — `PasswordAuthentication no`, `PermitRootLogin no`
- **Role:** `ansible/roles/calibre-importer/` (playbook `ansible/playbooks/calibre-import.yml`) — installs `calibre`, deploys the import script + a `calibre-import.timer` (polls every 2 min). Imports settled ebooks dropped into `/books-rw/_import` into the library via `calibredb`. See [Calibre-Web service doc → Auto-Import](../services/calibre-web.md#auto-import) for the CIFS/SQLite-locking constraint and the local-copy workaround.
- **Role:** `ansible/roles/docker-compose-update/` (playbook `ansible/playbooks/docker-compose-update.yml`, group `docker`) — updates the node's Compose stack(s) via `docker_compose_v2`: `pull: always` + `recreate: auto` (idempotent — recreates only on image/config change). Stack dirs in `host_vars/lxc220.yml` (`compose_projects`).

See: [Ansible Platform](../platform/ansible.md)
